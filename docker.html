El comando docker run -dit --name myApache -p 8080:80 httpd se utiliza para ejecutar un contenedor de Docker basado en la imagen "httpd", que es el servidor web Apache HTTP Server.

A continuación se explica brevemente cada parte del comando:

docker run: Es el comando principal de Docker utilizado para ejecutar un contenedor a partir de una imagen.
-dit: Son opciones o flags que se utilizan con el comando docker run:
-d indica que el contenedor se ejecutará en segundo plano (en modo daemon).
-i permite la interactividad al mantener abierto el canal de entrada estándar (stdin) del contenedor.
-t asigna un seudoterminal o pseudoTTY al contenedor.
--name myApache: Establece el nombre del contenedor como "myApache". Puedes elegir cualquier nombre que desees.
-p 8080:80: Mapea el puerto 8080 del host (el sistema operativo anfitrión en el que se está ejecutando Docker) al puerto 80 del contenedor. Esto permite acceder al servidor web Apache dentro del contenedor a través del puerto 8080 en el host.
httpd: Es el nombre de la imagen utilizada para crear el contenedor. En este caso, "httpd" es la imagen oficial de Apache HTTP Server.

En resumen, el comando se encarga de ejecutar un contenedor de Apache HTTP Server en segundo plano, asignándole un nombre, mapeando el puerto 8080 del host al puerto 80 del contenedor y utilizando la imagen "httpd". Esto te permitirá acceder al servidor web Apache ejecutándose dentro del contenedor a través del puerto 8080 de tu sistema.

Acceder a la terminal de un contenedor Docker en ejecución

❯ docker exec -it myApache bash

❯ root@fb489e16b2c8:/usr/local/apache2#

El comando docker exec -it myApache bash se utiliza para acceder a una terminal interactiva dentro de un contenedor de Docker en ejecución. Aquí hay una breve explicación de cada parte del comando:

  • docker exec: Es el comando de Docker utilizado para ejecutar comandos dentro de un contenedor en ejecución.
  • -it: Son opciones o flags que se utilizan con el comando docker exec:
    • -i permite la interactividad al mantener abierto el canal de entrada estándar (stdin) del contenedor.
    • -t asigna un seudoterminal o pseudoTTY al contenedor, lo que permite la interacción con la terminal.
  • myApache: Es el nombre o el ID del contenedor al que deseas acceder. Asegúrate de reemplazar "myApache" con el nombre o el ID correcto de tu contenedor.
  • bash: Es el comando que se ejecutará dentro del contenedor. En este caso, se utiliza "bash" para iniciar una sesión de terminal dentro del contenedor. Dependiendo de la imagen del contenedor, también se puede utilizar otro intérprete de comandos, como "sh" o "zsh".

Ubicación de los archivos para el renderizado en un contenedor de Apache HTTP Server (httpd)

Al ejecutar el comando docker exec -it myApache bash, se abrirá una terminal interactiva en el contenedor "myApache" y podrás interactuar con él como si estuvieras en una terminal normal. Esto te permitirá ejecutar comandos dentro del contenedor y realizar tareas dentro del entorno del contenedor.

Cuando ejecutas un contenedor basado en la imagen de Apache HTTP Server (httpd), por defecto los archivos para el renderizado de la página web se encuentran dentro del contenedor en el directorio /usr/local/apache2/htdocs.

Dentro de ese directorio, puedes colocar tus archivos HTML, CSS, JavaScript y cualquier otro recurso estático que desees servir a través del servidor web. Por ejemplo, si tienes un archivo HTML llamado index.html que deseas mostrar al visitar localhost:8080, debes colocar ese archivo dentro del directorio /usr/local/apache2/htdocs del contenedor.

❯ root@fb489e16b2c8:/usr/local/apache2# ls
bin  build  cgi-bin  conf  error  htdocs  icons  include  logs  modules

❯ root@fb489e16b2c8:/usr/local/apache2# cd htdocs/
❯ root@fb489e16b2c8:/usr/local/apache2/htdocs# ls
index.html

❯ echo '<h1>About</h1>' > about.html
❯ root@fb489e16b2c8:/usr/local/apache2/htdocs# ls
about.html  index.html

Ejecutar un servidor web Apache en Docker con montaje de archivos

El comando sudo docker run --name myWeb -dit -p 3000:80 -v "$PWD":/usr/local/apache2/htdocs/ httpd se utiliza para ejecutar un contenedor de Docker basado en la imagen "httpd" de Apache HTTP Server. Aquí hay una explicación de cada parte del comando:

  • sudo: Es un prefijo utilizado en sistemas Linux para ejecutar el comando con privilegios de superusuario (root), en caso de ser necesario.
  • docker run: Es el comando principal de Docker utilizado para ejecutar un contenedor a partir de una imagen.
  • --name myWeb: Establece el nombre del contenedor como "myWeb". Puedes elegir cualquier nombre que desees.
  • -dit: Son opciones o flags que se utilizan con el comando docker run:
    • -d indica que el contenedor se ejecutará en segundo plano (en modo daemon).
    • -i permite la interactividad al mantener abierto el canal de entrada estándar (stdin) del contenedor.
    • -t asigna un seudoterminal o pseudoTTY al contenedor.
  • -p 3000:80: Mapea el puerto 3000 del host (el sistema operativo anfitrión en el que se está ejecutando Docker) al puerto 80 del contenedor. Esto permite acceder al servidor web Apache dentro del contenedor a través del puerto 3000 en el host.
  • -v "$PWD":/usr/local/apache2/htdocs/: Monta el directorio actual (donde se ejecuta el comando) en el directorio /usr/local/apache2/htdocs/ dentro del contenedor. Esto permite que los archivos del directorio actual estén disponibles dentro del contenedor para el renderizado del servidor web.
  • httpd: Es el nombre de la imagen utilizada para crear el contenedor. En este caso, "httpd" es la imagen oficial de Apache HTTP Server.

Detener todos los contenedores en Docker

El comando docker stop $(docker ps -aq) se utiliza para detener todos los contenedores en ejecución en Docker. Aquí está una explicación de cada parte del comando:

  • docker ps: Muestra información sobre los contenedores en ejecución.
  • -a: Muestra todos los contenedores, incluidos los que están detenidos.
  • -q: Solo muestra las IDs de los contenedores.

El comando docker ps -aq se utiliza para obtener una lista de las IDs de todos los contenedores, ya sean contenedores en ejecución o detenidos.

La construcción $(docker ps -aq) se conoce como "command substitution" (sustitución de comandos). Básicamente, esto toma el resultado de docker ps -aq (las IDs de los contenedores) y lo pasa como argumento al comando docker stop.

Por lo tanto, docker stop $(docker ps -aq) ejecutará el comando docker stop con las IDs de todos los contenedores, lo que resultará en detener todos los contenedores en ejecución.

docker stop <nombreContenedor> o bien docker stop <idContenedor>

❯ docker stop myApache

Cuando ejecutas docker ps -a, se mostrará una lista de todos los contenedores en Docker junto con su información relevante, como el nombre o el ID del contenedor, la imagen utilizada, el estado (en ejecución o detenido), los puertos expuestos y la fecha de creación, entre otros detalles.

Este comando es útil para obtener una visión general de todos los contenedores presentes en tu entorno Docker, independientemente de su estado. Puedes utilizar esta información para administrar y realizar acciones específicas en los contenedores, como iniciar, detener o eliminarlos.

Eliminar todos los contenedores en Docker

El comando docker rm $(docker ps -aq) se utiliza para eliminar todos los contenedores en Docker, tanto los que están en ejecución como los que están detenidos. Aquí tienes una explicación de cada parte del comando:

  • docker ps: Muestra información sobre los contenedores en ejecución.
  • -a: Muestra todos los contenedores, incluidos los que están detenidos.
  • -q: Solo muestra las IDs de los contenedores.

El comando docker ps -aq se utiliza para obtener una lista de las IDs de todos los contenedores, ya sean contenedores en ejecución o detenidos.

La construcción $(docker ps -aq) se conoce como "command substitution" (sustitución de comandos). Básicamente, esto toma el resultado de docker ps -aq (las IDs de los contenedores) y lo pasa como argumento al comando docker rm.

Por lo tanto, docker rm $(docker ps -aq) ejecutará el comando docker rm con las IDs de todos los contenedores, lo que resultará en la eliminación de todos los contenedores en Docker.

Es importante tener en cuenta que esta acción eliminará permanentemente los contenedores y toda la información asociada a ellos. Asegúrate de tener una copia de seguridad de cualquier dato importante antes de ejecutar este comando.

Construir una imagen de Docker con Apache y un sitio web personalizado

  1. Debes crear un sitio web con archivo index, css, javascript, etc
  2. Crear archivo docker Dockerfile e ingresar estos comandos en su interior
    FROM httpd
    COPY ./public_html /usr/local/apache2/htdocs/
  1. La estructura quedaría algo así:
    WebPage/
    ├── public_html/
    │   ├── index.html
    │   ├── css/
    │   │   ├── style.css
    │   ├── js/
    │   │   ├── script.js
    │   ├── images/
    │   │   ├── image1.jpg
    │   │   ├── image2.png
    ├── Dockerfile
  1. Ejecuta el siguiente comando
    ❯ docker build -t docker-apache .

El comando build -t docker-apache . se utiliza para construir una imagen de Docker a partir de un Dockerfile y asignarle un nombre o etiqueta. Aquí tienes una explicación de cada parte del comando:

  • docker build: Es el comando utilizado para construir una imagen de Docker.
  • -t: Es una opción que se utiliza para asignarle un nombre o etiqueta a la imagen.
  • docker-apache: Es el nombre o etiqueta que le asignamos a la imagen. Puedes elegir el nombre que desees, siempre y cuando sigas las convenciones de nombres de Docker.
  • .: Es el contexto de construcción. Aquí especificamos que el Dockerfile y cualquier archivo necesario se encuentran en el directorio actual.

En cuanto al contenido del Dockerfile, FROM httpd indica que estamos utilizando la imagen base de Apache HTTP Server como punto de partida para construir nuestra imagen personalizada. COPY ./public_html /usr/local/apache2/htdocs/ copia el contenido del directorio public_html local al directorio /usr/local/apache2/htdocs/ dentro del contenedor.
