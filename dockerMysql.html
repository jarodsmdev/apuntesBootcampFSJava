Crear container de MySQL

❯ docker run --name mysql-app -d -p 3307:3306 --hostname mysql-app -e MYSQL_ROOT_PASSWORD=Password2023$ mysql:latest

Este comando se utiliza para ejecutar un contenedor de Docker basado en la imagen de MySQL con la configuración especificada, el comando se explica a continuación:

  • docker run: Comando utilizado para ejecutar un contenedor.
  • --name mysql-app: Establece el nombre del contenedor como "mysql-app".
  • -d: Ejecuta el contenedor en segundo plano (modo desacoplado).
  • -p 3307:3306: Mapea el puerto 3307 de la máquina anfitriona (host) al puerto 3306 del contenedor. Esto permite acceder al servidor MySQL dentro del contenedor utilizando el puerto 3307 en la máquina anfitriona.
  • --hostname mysql-app: Establece el nombre de host del contenedor como "mysql-app". Tenga en cuenta que esta opción no afecta el nombre de host dentro del contenedor, sino solo el nombre de host que se resuelve en la máquina anfitriona.
  • -e MYSQL_ROOT_PASSWORD=Password2023$: Establece la variable de entorno MYSQL_ROOT_PASSWORD con el valor "Password2023$". Esta opción configura la contraseña de root para el servidor MySQL dentro del contenedor.
  • mysql:latest: Especifica el nombre de la imagen de Docker a utilizar, en este caso, "mysql:latest". Esto descargará la imagen más reciente de MySQL desde Docker Hub si no está presente localmente.

Creación de usuario administrador en MySQL con privilegios globales

mysql> CREATE USER 'admin'@'%' IDENTIFIED BY 'AdminGeneral2023$';
Query OK, 0 rows affected (0.16 sec)
  • CREATE USER: Es una declaración en MySQL utilizada para crear un nuevo usuario.
  • 'admin'@'%': Es la especificación del nombre de usuario y el host desde el cual se permite la conexión. En este caso, se está creando un usuario llamado "admin" que puede conectarse desde cualquier host ('%').
  • IDENTIFIED BY 'AdminGeneral2023$': Es la cláusula utilizada para establecer la contraseña del usuario. En este caso, se establece la contraseña como "AdminGeneral2023$".

Consultar Usuarios de MySQL

❯ mysql> SELECT USER, plugin FROM mysql.user;

| mysql.session    | caching_sha2_password |
| mysql.sys        | caching_sha2_password |
| root             | caching_sha2_password |
+------------------+-----------------------+
6 rows in set (0.00 sec)

El comando anterior se utiliza en MySQL para consultar la información de usuarios y plugins asociados en la tabla mysql.user. A continuación, te explico cada parte del comando:

  • SELECT: Es una declaración en MySQL utilizada para seleccionar y mostrar datos de una tabla o vista.
  • USER, plugin: Son las columnas que se seleccionan para mostrar en el resultado de la consulta. En este caso, se seleccionan las columnas USER (nombre de usuario) y plugin (plugin asociado al usuario).
  • FROM mysql.user: Es la cláusula que especifica la tabla de la cual se seleccionan los datos. En este caso, se selecciona la tabla mysql.user.

Conceder permisos al usuario recién creado

Otorgar todos los privilegios a un usuario específico en todas las bases de datos y tablas del servidor.

❯ mysql> GRANT ALL PRIVILEGES ON *.* TO 'admin'@'%';
Query OK, 0 rows affected (0.09 sec)
  • GRANT ALL PRIVILEGES: Es una declaración en MySQL utilizada para otorgar todos los privilegios disponibles en el servidor.
  • ON *.*: Es la cláusula que especifica a qué bases de datos y tablas se aplicarán los privilegios. En este caso, *.* indica que se aplicarán a todas las bases de datos y tablas.
  • TO 'admin'@'%': Es la cláusula que indica el nombre de usuario y el host desde el cual se permite la conexión. En este caso, se otorgan los privilegios al usuario "admin" que puede conectarse desde cualquier host ('%').

Este comando otorga todos los privilegios disponibles, como SELECT, INSERT, UPDATE, DELETE, etc., al usuario "admin" para todas las bases de datos y tablas del servidor de MySQL. El usuario "admin" tendrá pleno acceso y control sobre todas las operaciones en las bases de datos y tablas del servidor.

Verficar Permisos de un usuario MySQL

❯ mysql> SHOW GRANTS FOR 'admin'@'%';

Actualizar privilegios entregados

❯ mysql> FLUSH PRIVILEGES;

Iniciar sesión en MySQL con el usuario Creado

❯ bash-4.4# mysql -u admin -pAdminGeneral2023$
