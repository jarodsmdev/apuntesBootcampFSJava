Crearemos un container

❯ docker run --name  postgres-app -e POSTGRES_PASSWORD=EstoEsUnaContrasenia2023$ -d -p 5433:5432 postgres:latest
a568c49de7bafbc3117373fb30ea27c965958cc28d24e4fdbb0ee959009b4cd1

❯ docker ps
CONTAINER ID   IMAGE             COMMAND                  CREATED          STATUS          PORTS                                       NAMES
a568c49de7ba   postgres:latest   "docker-entrypoint.s…"   53 seconds ago   Up 50 seconds   0.0.0.0:5433->5432/tcp, :::5433->5432/tcp   postgres-app
  • docker run: Es el comando principal de Docker para ejecutar un contenedor a partir de una imagen.
  • --name postgres-app: Establece un nombre para el contenedor, en este caso, "postgres-app". Este nombre se puede utilizar posteriormente para referirse al contenedor en otros comandos Docker.
  • -e POSTGRES_PASSWORD=Password2023$: Especifica una variable de entorno dentro del contenedor. En este caso, se está configurando la contraseña de PostgreSQL con el valor "Password2023$". Esta contraseña se utilizará para acceder a la base de datos PostgreSQL dentro del contenedor.
  • -d: Esta opción indica que el contenedor se ejecutará en segundo plano (modo "detach").
  • -p 5433:5432: Establece la asignación de puertos, donde el primer número (5433) representa el puerto del host local y el segundo número (5432) representa el puerto dentro del contenedor donde se encuentra PostgreSQL. En este caso, el puerto 5433 del host local se mapea al puerto 5432 dentro del contenedor.
  • postgres:latest: Especifica la imagen de Docker a utilizar para crear el contenedor. En este caso, se utiliza la imagen "postgres:latest", que es la última versión estable de PostgreSQL disponible en Docker Hub.

Creación de un nuevo rol de superusuario en PostgreSQL con atributos y privilegios

❯ docker ps 
CONTAINER ID   IMAGE             COMMAND                  CREATED          STATUS          PORTS                                       NAMES
a568c49de7ba   postgres:latest   "docker-entrypoint.s…"   32 minutes ago   Up 32 minutes   0.0.0.0:5433->5432/tcp, :::5433->5432/tcp   postgres-app

❯ docker exec -it postgres-app bash
root@a568c49de7ba:/# psql -U postgres
psql (15.3 (Debian 15.3-1.pgdg120+1))
Type "help" for help.

postgres=# CREATE ROLE admin WITH
postgres-# LOGIN
postgres-# SUPERUSER
postgres-# CREATEDB
postgres-# CREATEROLE
postgres-# INHERIT
postgres-# REPLICATION
postgres-# CONNECTION LIMIT -1
postgres-# PASSWORD 'Password2023%';
CREATE ROLE
postgres=#

Este comando crea un nuevo rol de usuario llamado "admin" con privilegios de superusuario, la capacidad de iniciar sesión, crear bases de datos y roles, heredar privilegios, participar en la replicación y sin límite de conexiones. También se establece una contraseña para el rol.

  1. CREATE ROLE admin WITH: Inicia la creación de un nuevo rol de usuario llamado "admin". El comando CREATE ROLE se utiliza para crear un nuevo rol en PostgreSQL.

  2. LOGIN: Este atributo indica que el rol de usuario puede iniciar sesión en PostgreSQL. Los usuarios con este atributo pueden autenticarse y acceder a la base de datos.

  3. SUPERUSER: Este atributo otorga privilegios de superusuario al rol. Los superusuarios tienen todos los privilegios y pueden realizar cualquier acción en el sistema.

  4. CREATEDB: Este atributo permite al rol de usuario crear nuevas bases de datos. Los usuarios con este atributo pueden utilizar el comando CREATE DATABASE para crear bases de datos adicionales.

  5. CREATEROLE: Este atributo permite al rol de usuario crear nuevos roles de usuario. Los usuarios con este atributo pueden utilizar el comando CREATE ROLE para crear roles adicionales.

  6. INHERIT: Este atributo indica que el rol de usuario hereda los privilegios y configuraciones del rol padre. Si un rol de usuario tiene este atributo, heredará los privilegios del rol especificado como su rol padre.

  7. REPLICATION: Este atributo permite al rol de usuario participar en la replicación de datos en PostgreSQL. Los usuarios con este atributo pueden tener un rol en la configuración de la replicación.

  8. CONNECTION LIMIT -1: Este atributo establece el límite de conexiones para el rol de usuario. En este caso, se establece en -1, lo que significa que no hay límite de conexiones para el rol.

  9. PASSWORD 'Password2023%': Esta línea establece la contraseña para el rol de usuario. Reemplaza "Password2023%" con la contraseña real que deseas establecer para el rol.

Creación de una base de datos en PostgreSQL con propiedades

postgres=# CREATE DATABASE db_inmueble
postgres-# WITH
postgres-# OWNER = admin
postgres-# ENCODING = 'UTF-8'
postgres-# CONNECTION LIMIT = -1;
CREATE DATABASE
postgres=#

Este comando crea una nueva base de datos llamada "db_inmueble" en PostgreSQL, asigna el rol de usuario "admin" como propietario, establece la codificación de caracteres como UTF-8 y no impone un límite en el número de conexiones a la base de datos.

CREATE DATABASE se utiliza en PostgreSQL para crear una nueva base de datos con configuraciones personalizadas. A continuación, se presenta una breve descripción de las opciones utilizadas en el comando:

  • db_inmueble: Especifica el nombre de la base de datos a crear. En este caso, la base de datos se llama "db_inmueble".
  • OWNER = admin: Establece el propietario de la base de datos como el rol de usuario "admin". El rol "admin" será el propietario y tendrá privilegios de control sobre la base de datos.
  • ENCODING = 'UTF-8': Establece la codificación de caracteres de la base de datos como UTF-8. Esto permite almacenar y manipular datos en varios idiomas y caracteres especiales.
  • CONNECTION LIMIT = -1: Define el límite de conexiones para la base de datos. En este caso, se establece en -1, lo que significa que no hay límite de conexiones para la base de datos.

Iniciar sesión con el usuario admin a la bd db_inmueble recién creados

psql -d db_inmueble -U admin
psql (15.3 (Debian 15.3-1.pgdg120+1))
Type "help" for help.

db_inmueble=#

El código psql -d db_inmueble -U admin se utiliza para abrir una sesión interactiva en PostgreSQL, conectándose a la base de datos "db_inmueble" y utilizando el usuario "admin" para la autenticación. Una vez ejecutado este comando, se abrirá la interfaz de línea de comandos de PostgreSQL, donde podrás ingresar y ejecutar comandos SQL relacionados con la base de datos especificada.

Explicación de cada parte del código:

  • psql: Es el comando que se utiliza para iniciar la interfaz de línea de comandos de PostgreSQL, llamada "psql". Permite ejecutar comandos SQL y administrar bases de datos desde la terminal.

  • -d db_inmueble: Especifica la base de datos a la cual se desea acceder. En este caso, se indica que se quiere conectar a la base de datos llamada "db_inmueble".

  • -U admin: Indica el nombre de usuario con el cual se desea iniciar sesión en PostgreSQL. En el ejemplo, se utiliza el usuario "admin" para autenticarse en el servidor de PostgreSQL.

Creación de un esquema en una base de datos PostgreSQL con autorización

db_inmueble=# CREATE SCHEMA sc_inmueble
db_inmueble-# AUTHORIZATION admin;
CREATE SCHEMA
db_inmueble=#

El comando CREATE SCHEMA se utiliza en PostgreSQL para crear un nuevo esquema dentro de una base de datos. A continuación, se presenta una breve descripción del comando y su ejecución en el ejemplo proporcionado:

CREATE SCHEMA sc_inmueble AUTHORIZATION admin;

  • CREATE SCHEMA sc_inmueble: Indica la creación de un nuevo esquema llamado "sc_inmueble". El esquema es una estructura lógica que se utiliza para organizar y agrupar objetos relacionados, como tablas, vistas y funciones, dentro de una base de datos en PostgreSQL.

  • AUTHORIZATION admin: Establece al usuario "admin" como el propietario del esquema "sc_inmueble". El usuario "admin" será responsable de administrar y controlar los objetos dentro de este esquema.

Listar esquemas en una base de datos PostgreSQL

db_inmueble=# \dn
        List of schemas
    Name     |       Owner       
-------------+-------------------
public      | pg_database_owner
sc_inmueble | admin
(2 rows)

El comando \dn se utiliza en PostgreSQL para listar los esquemas (schemas) existentes en una base de datos. A continuación, se presenta la explicación del resultado obtenido en el ejemplo proporcionado:

  • List of schemas: Esta línea indica que a continuación se mostrará una lista de esquemas.

  • Name: Columna que muestra el nombre de cada esquema.

  • Owner: Columna que indica el propietario de cada esquema.

  • public: Es el nombre del esquema predeterminado en PostgreSQL y generalmente contiene los objetos por defecto.

  • sc_inmueble: Es el nombre de otro esquema existente en la base de datos. En este caso, el esquema "sc_inmueble" tiene como propietario al usuario "admin".

Establecer el esquema de búsqueda predeterminado en PostgreSQL

SET search_path TO sc_inmueble;

Indica que estás configurando el esquema de búsqueda predeterminado en PostgreSQL para que todas las consultas posteriores se realicen en el esquema "sc_inmueble" sin necesidad de especificarlo en cada consulta.
